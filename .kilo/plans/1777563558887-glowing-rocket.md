# 計劃：圖片連結外顯、隱藏 Group Commission、URL Hash 路由、GitHub Pages 部署

## 需求總覽

1. **item.description 外顯**：在 GalleryPanel 圖片與 dot navigation 之間顯示當前作品的 `item.description`，包含 HTML 超連結。
2. **隱藏 group 的 commission**：group 頁面只顯示 works 與 contact，不顯示 commission tab。
3. **URL hash 路由**：讓 `wormbakery.com#personal` 與 `wormbakery.com#group` 可以直接導向對應身份，切換時同步更新 hash。
4. **GitHub Pages 部署**：設定 Next.js 靜態輸出與 GitHub Actions workflow。

---

## 1. item.description 外顯（圖片與導航之間）

### 現況

- `item.description` 目前只在 `Lightbox` 裡顯示（第 836 行）
- GalleryPanel（主畫面）只顯示圖片、dot、上下箭頭，沒有描述文字

### 修改方向

- 在 `GalleryPanel`（profile-page.tsx）的圖片區塊（`</motion.button>` 後）與 dot navigation 區塊之間，插入一個 `item.description` 的區塊
- 使用 `dangerouslySetInnerHTML` 渲染 HTML，並套用與 Lightbox 一致的連結樣式（`[&_a]:underline` 等）
- 移除 Lightbox 裡的 `item.description`（避免重複顯示）
- 或者保留 Lightbox 的描述也行（點開大圖仍可看到），但主畫面必須也能看到

**結論**：主畫面新增描述 + Lightbox 保留不動（雙重可見性較好）

## 2. 隱藏 group 的 commission tab

### 現況

- `TAB_ORDER = ["works", "commission", "contact"]`（第 84 行）
- `SectionTabs` 直接根據 `TAB_ORDER` 渲染所有 tab，無條件判斷
- group.sections.commission 在資料檔中仍然存在

### 修改方向

**選項 A**：在 `SectionTabs` 裡直接判斷 identity，如果是 group 就不渲染 commission  
**選項 B**：改由每個 profile 定義自己的 `tabOrder`

**結論**：選項 A 較簡單，改動最小。但會造成型別上 `GalleryTab` 仍包含 commission。若要更乾淨，改為讓 `ProfileContentEntry` 自行定義可見的 tabs。

最終選用 **選項 A 的變體**：在 `ProfilePage` 中定義 `visibleTabs`，根據 identity 動態計算，再傳入 `SectionTabs`。

```ts
const visibleTabs = identity === "group"
  ? (["works", "contact"] as const)
  : (["works", "commission", "contact"] as const);
```

這樣 SectionTabs 渲染時就不會出現 commission，而且 Contact 的顯示邏輯也完全正常。

## 3. URL Hash 路由

### 現況

- `identity` 使用 `useState("personal")`，沒有讀取 URL hash

### 修改方向

1. **進入頁面讀取 hash**：在 `useEffect`（空依賴）中讀取 `window.location.hash`，解析 `#personal` 或 `#group`，設定初始 `identity`
2. **切換時更新 hash**：在 `handleIdentityChange` 中使用 `window.history.replaceState` 更新 URL hash
3. **hashchange 監聽**：可選，監聽瀏覽器的 back/forward 按鈕

```ts
// 初始化
useEffect(() => {
  const hash = window.location.hash.replace("#", "") as IdentityKey;
  if (hash === "personal" || hash === "group") {
    setIdentity(hash);
  }
}, []);

// 切換時更新
const handleIdentityChange = (nextIdentity: IdentityKey) => {
  window.history.replaceState(null, "", `#${nextIdentity}`);
  // ... 原有邏輯
};
```

## 4. GitHub Pages 靜態部署

### 現況

- `next.config.mjs` 已有 `images: { unoptimized: true }`，但沒有設定 `output: 'export'`
- 沒有 `.github/workflows/deploy.yml` 或 GitHub Actions 目錄

### 修改方向

1. **next.config.mjs**：加入 `output: 'export'` 與 `distDir: 'dist'`
2. **建立 GitHub Actions workflow**：`.github/workflows/deploy.yml`
   - trigger: `push` to `main`
   - steps: checkout → install → build → upload pages artifact → deploy
3. **GitHub Pages 設定**：在 repo Settings > Pages 選擇「GitHub Actions」作為 source

由於使用**自訂網域**（wormbakery.com），**不需要 `basePath`**。

workflow 範例：
```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist
  deploy:
    needs: build
    runs-on: ubuntu-latest
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/deploy-pages@v4
        id: deployment
```
