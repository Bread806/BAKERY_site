# 計劃：修正 Works / Commission 圖片排版並移除右側頁碼欄位

## 現況分析

在 `components/profile/profile-page.tsx` 的 `GalleryPanel` 元件中（約第 592–655 行）：

1. **兩欄 layout**：使用 `lg:grid-cols-[minmax(0,1fr)_minmax(120px,0.24fr)]`，左邊放圖片、右邊放頁碼與切換按鈕。
2. **圖片被裁切**：圖片容器寫死 `aspect-[16/9] min-h-[11rem] sm:min-h-[15rem] md:min-h-[18rem]`，加上 `fill` 模式與 `sizes="(min-width: 1024px) 60vw, 100vw"`，導致不同比例的圖片被強制切成 16:9。
3. **右側頁碼欄位**：顯示 `01 / 03` 以及 mobile 用的 `< >` 按鈕。

## 修改方向

### 1. 移除右側頁碼欄位
刪除 `GalleryPanel` 中整個右側區塊（約第 635–653 行），包含：
- 頁碼文字 `01 / 03`
- mobile 用的 `IconButton` 上下頁切換（原本放在右側欄位底部）

### 2. 圖片容器改為單欄滿寬
- 將外層 `grid gap-3 lg:grid-cols-[...]` 改為 `grid grid-cols-1 gap-3`
- 讓圖片按鈕獨占整列寬度

### 3. 圖片完整顯示（不再裁切）
- 移除固定 `aspect-[16/9]` 與 `min-h-*` 限制
- 改用 Next.js `<Image>` 的 `width`、`height` 屬性（取用 `item.image.width / item.image.height`），讓圖片依原始比例自然呈現
- `className` 改為 `"h-auto w-full"`，維持 `object-contain` / `object-cover` 的既有邏輯
- 移除原本包裹的 `absolute inset-0` 定位層，改為 `relative w-full` 的流式佈局

### 4. 移動上下頁切換按鈕到 dot indicator 旁邊
- 原本 desktop 的 `< >` 按鈕在標題旁（第 582–589 行，className `hidden` on mobile），改為 `flex`（即不限 `md` 以上才顯示）
- 在最下方 dot indicator 的左右兩側各放一顆 `< >` 按鈕，供所有裝置點擊
- 調整為 `flex items-center justify-center gap-3`（或合適的 gap），使 dot indicator 與按鈕水平排列
- 上方標題區原本的 `< >` 按鈕保持不動，但改為僅 `hidden md:flex`（避免重複）

## 預計修改檔案

- `components/profile/profile-page.tsx`
