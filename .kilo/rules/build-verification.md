# 修改後必須驗證 Build

每次完成程式碼修改後，在回報完成之前，**必須**執行以下指令確認專案可以正常運作：

```bash
npm run build
```

## 執行規則

- 不論修改範圍大小（就算只改一行 class），都要跑 build
- Build **成功**才能回報任務完成
- Build **失敗**必須自行排查錯誤並修復，直到 build 成功為止
- 不可以把 build 失敗的狀態交給使用者處理

## 回報格式

任務完成時，回報內容需包含：

```
✅ Build 成功
- 修改內容：[簡短說明改了什麼]
- 影響檔案：[列出修改的檔案]
```

若 build 過程中有 warning，需一併列出讓使用者知道。
